Kapitel 13: Virtualisierung

Mehrere virtuelle Maschinen auf einer Hardware.

Virtualisierung

Virtualisierung bezeichnet eine Technologie, die es ermöglicht, die Hardware-Ressourcen eines einzelnen Computers (CPU, Speicher, Festplatte, Netzwerk) so aufzuteilen, dass darauf mehrere voneinander isolierte virtuelle Maschinen (VMs) laufen können. Jede VM verhält sich wie ein eigenständiger Computer und kann ihr eigenes Betriebssystem (genannt Gast-Betriebssystem) ausführen.

Die Software, die diese Virtualisierung ermöglicht und die VMs verwaltet, wird als Hypervisor oder Virtual Machine Monitor (VMM) bezeichnet.

Vorteile der Virtualisierung

  • Serverkonsolidierung: Mehrere physische Server können auf einem einzigen Server als VMs zusammengefasst werden. Das spart Kosten für Hardware, Strom und Kühlung.
  • Effiziente Ressourcennutzung: Die Hardware-Auslastung wird deutlich verbessert, da Ressourcen dynamisch den VMs zugewiesen werden können, die sie gerade benötigen.
  • Isolation und Sicherheit: VMs sind stark voneinander isoliert. Ein Absturz oder ein Sicherheitsproblem in einer VM beeinträchtigt die anderen VMs nicht.
  • Flexibilität und Portabilität: Eine VM kann als einfache Datei gespeichert, kopiert und auf eine andere physische Maschine migriert werden, oft sogar im laufenden Betrieb.
  • Test- und Entwicklungsumgebungen: Entwickler können schnell neue VMs mit unterschiedlichen Betriebssystemen und Konfigurationen aufsetzen, um Software zu testen, ohne ihre physischen Maschinen zu verändern.

Typen von Hypervisoren

Typ-1-Hypervisor (Nativ / Bare-Metal)

Ein Typ-1-Hypervisor läuft direkt auf der Hardware des Host-Systems, genau wie ein reguläres Betriebssystem. Er ist selbst der Kernel und verwaltet die Hardware direkt. Alle Gast-Betriebssysteme laufen auf einer Abstraktionsebene über dem Hypervisor.

  • Vorteile: Sehr hohe Leistung und Effizienz, da es keine zusätzliche Betriebssystemschicht zwischen Hypervisor und Hardware gibt.
  • Beispiele: VMware ESXi, Microsoft Hyper-V, Xen.

Typ-2-Hypervisor (Hosted)

Ein Typ-2-Hypervisor ist eine Anwendung, die auf einem vollwertigen Host-Betriebssystem (z.B. Windows, macOS oder Linux) installiert wird. Die Gast-Betriebssysteme laufen dann als Prozesse innerhalb des Host-Betriebssystems.

  • Vorteile: Einfach zu installieren und zu verwenden, ideal für Desktops und Entwickler.
  • Nachteile: Geringere Leistung als Typ-1, da jeder Zugriff auf die Hardware durch eine zusätzliche Betriebssystemschicht (den Host) gehen muss.
  • Beispiele: Oracle VirtualBox, VMware Workstation/Fusion, Parallels Desktop.

Hardware-Unterstützung für Virtualisierung

Moderne CPUs (z.B. mit Intel VT-x oder AMD-V Technologie) bieten spezielle Befehlssätze, die dem Hypervisor helfen, die VMs effizienter zu verwalten. Diese Hardware-Unterstützung reduziert den Overhead erheblich, der durch die Virtualisierung von privilegierten Befehlen entsteht, und macht die Ausführung von VMs fast so schnell wie die native Ausführung auf der Hardware.
